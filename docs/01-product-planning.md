# 01. Product Planning

## 1) 프로젝트 개요

- 프로젝트명: `algoitni`
- 한 줄 설명: 대표 파이썬 구현 1개를 기준으로 알고리즘 개념 학습부터 반례 기반 오답 진단까지 연결하는 학습 서비스
- 개발 맥락: docs-first, 데모 우선 워크숍 MVP
- 팀 구성: 솔로 개발 기준

## 2) 문제 정의

- 현재 불편함: 알고리즘 학습자가 개념은 읽었지만 구현 순서와 상태 변화를 체화하지 못한 채 실전 문제에서 바로 막힌다
- 왜 지금 풀 가치가 있는가: 입문자에게 필요한 것은 예제 양이 아니라 정답 구조가 고정된 작은 학습 흐름과 오답 원인 설명이다
- 기존 방식의 한계:
  - 개념 설명과 문제 풀이 사이에 학습 브리지가 약하다
  - 여러 대표 예제를 나열하면 초심자가 무엇을 기준 코드로 잡아야 할지 헷갈린다
  - 오답 결과가 `틀렸습니다`로 끝나면 반례를 찾기 전까지 학습이 멈춘다

## 3) 타겟 사용자

- 핵심 사용자: 알고리즘 입문자
- 사용 상황: 특정 알고리즘을 막 배웠거나, 구현 순서와 조건 분기를 반복해서 헷갈릴 때
- 사용 맥락: 데스크톱 중심의 단기 집중 학습, 모바일 보조 확인

## 4) 목표

- 제품 목표: 알고리즘마다 대표 파이썬 구현 1개를 기준으로 개념 이해, 단계형 연습, 실전 적용을 하나의 흐름으로 제공한다
- 사용자 경험 목표: 각 단계에서 `합격 / 재도전`이 명확하고, 틀렸을 때 즉시 짧은 이유와 다음 시도 방향을 얻도록 한다
- 구현 목표: mock-first가 아니라 정적 콘텐츠 + 실제 Flask API를 사용하는 lightweight real-backend MVP를 만든다

반드시 구현할 것:

- [x] 알고리즘별 대표 파이썬 구현 레슨 1개
- [x] `빈칸 -> 파슨스` 2단계 deterministic 연습
- [x] 힌트, 재설명, 오답 분석을 위한 LLM 보조 API
- [x] 실전 문제 1개와 반례 생성 파이프라인

## 7) 핵심 사용자 흐름

### Flow A. 대표 구현 레슨 학습

1. 사용자가 홈에서 알고리즘을 선택한다.
2. 개념 설명 화면에서 핵심 아이디어, 복잡도, 시각화를 확인한다.
3. 대표 파이썬 구현 레슨으로 이동한다.
4. `빈칸` 단계를 수행하고 deterministic 채점을 받는다.
5. `빈칸`을 통과하면 `파슨스` 단계가 해금된다.
6. `파슨스`를 통과하면 레슨 요약과 실전 주의 포인트를 확인한다.

### Flow B. 학습 보조 요청

1. 사용자가 개념 또는 레슨 화면에서 힌트, 재설명, 오답 분석을 요청한다.
2. 시스템은 정적 힌트를 우선 사용하고, 필요 시 LLM으로 더 쉬운 표현이나 오답 원인을 설명한다.
3. 사용자는 피드백을 바탕으로 다시 시도한다.

### Flow C. 실전 문제와 반례 확인

1. 사용자가 실전 문제 화면에서 코드를 입력하고 제출 또는 실행을 시도한다.
2. 실패하거나 확신이 없으면 반례 생성을 요청한다.
3. 시스템은 known edge case, reference solver, brute force, LLM 후보 생성을 순차적으로 사용한다.
4. mismatch가 검증되면 입력, 사용자 출력, 기대 출력을 함께 보여준다.

## 8) 성공 기준

- [x] 알고리즘 하나를 기준으로 개념 -> 대표 구현 레슨 -> 실전 문제 흐름이 끊기지 않는다
- [x] `빈칸`과 `파슨스` 합격 판정이 deterministic 규칙으로만 동작한다
- [x] LLM은 힌트, 재설명, 오답 분석, 반례 후보 생성 보조에만 사용된다
- [x] 기존 다중 예제 중심 구조가 문서와 데이터 모델에서 제거된다
- [x] README와 `docs/01~06`이 같은 MVP 경계를 유지한다

## 10) Post-MVP 확장 방향

### Phase 1. 학습 단계 확장

- 백지 코딩 단계 복구
- 사용자 오답 패턴 분석
- 브리지 체크 문제 고도화

### Phase 2. 운영 확장

- 로그인 및 진도 동기화
- 알고리즘 수 확장
- 반례 최소화 고도화

확장 원칙:

- deterministic 채점 경계를 흐리는 기능은 MVP 이후에도 별도 계약 없이 넣지 않는다
- 대표 구현 1개 중심 구조를 깨는 확장은 사용자 혼란 대비 가치가 검증될 때만 추가한다
